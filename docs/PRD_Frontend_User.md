## 1. 개요

본 문서는 OrderBean 서비스의 **사용자(Customer) 화면** UI/UX 요구사항을 정의합니다. 사용자 화면은 고객이 카페 메뉴를 조회하고, 장바구니에 담고, 주문을 완료하는 핵심 플로우를 담당합니다.

---

## 2. 화면 구성

### 2.1 전체 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│  [로고: COZY]              [주문하기]  [관리자]         │  ← 헤더 (Navbar)
├─────────────────────────────────────────────────────────┤
│                                                         │
│                    메인 콘텐츠 영역                      │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                    장바구니 영역                         │  ← 하단 고정 (선택적)
└─────────────────────────────────────────────────────────┘
```

### 2.2 헤더 (Navbar)

| 요소 | 설명 | 동작 |
|------|------|------|
| 로고 (COZY) | 카페 브랜드명 표시 | 클릭 시 홈/메뉴 화면으로 이동 |
| 주문하기 버튼 | 주문 페이지 링크 | 클릭 시 주문 내역 화면으로 이동 |
| 관리자 버튼 | 관리자 모드 접근 | 클릭 시 관리자 대시보드로 이동 |

**스타일:**
- 배경: 흰색 (#FFFFFF)
- 테두리: 하단 1px 회색 보더
- 높이: 60px
- 로고: 좌측 정렬, 폰트 굵게
- 버튼: 우측 정렬, 테두리 스타일

## 3. 메뉴 화면 (Menu Page)

### 3.1 화면 목적
사용자가 카페의 메뉴를 조회하고 원하는 메뉴를 장바구니에 담을 수 있는 화면

### 3.2 메뉴 카드 컴포넌트

```
┌─────────────────────┐
│   ┌─────────────┐   │
│   │             │   │
│   │   이미지    │   │  ← 메뉴 이미지 (placeholder 또는 실제 이미지)
│   │     ✕      │   │
│   │             │   │
│   └─────────────┘   │
│                     │
│  메뉴명             │  ← 예: 아메리카노(ICE)
│  가격               │  ← 예: 4,000원
│                     │
│  간단한 설명...     │  ← 메뉴 설명 (truncated)
│                     │
│  □ 샷 추가 (+500원) │  ← 옵션 체크박스
│  □ 시럽 추가 (+0원) │  ← 옵션 체크박스
│                     │
│  ┌─────────────┐    │
│  │    담기     │    │  ← 장바구니 추가 버튼
│  └─────────────┘    │
└─────────────────────┘
```

### 3.3 메뉴 카드 상세 스펙

| 요소 | 스펙 |
|------|------|
| 카드 크기 | 가로 약 200px, 높이 자동 |
| 이미지 영역 | 정사각형, 회색 배경 + X 표시 (placeholder) |
| 메뉴명 | 폰트: 16px, 굵게, 검정색 |
| 가격 | 폰트: 14px, 회색 |
| 설명 | 폰트: 12px, 회색, 최대 2줄 |
| 옵션 | 체크박스 + 라벨 (가격 포함) |
| 담기 버튼 | 테두리 스타일, hover 시 배경 변경 |

### 3.4 메뉴 그리드 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│  ┌─────────┐    ┌─────────┐    ┌─────────┐              │
│  │ 아메리  │    │ 아메리  │    │ 카페    │              │
│  │ 카노    │    │ 카노    │    │ 라떼    │              │
│  │ (ICE)   │    │ (HOT)   │    │         │              │
│  │ 4,000원 │    │ 4,000원 │    │ 5,000원 │              │
│  │         │    │         │    │         │              │
│  │ [담기]  │    │ [담기]  │    │ [담기]  │              │
│  └─────────┘    └─────────┘    └─────────┘              │
└──────────────────────────────────────────────────────────┘
```

**그리드 스펙:**
- 데스크톱: 3열 그리드
- 태블릿: 2열 그리드
- 모바일: 1열 그리드
- 카드 간격: 20px

### 3.5 메뉴 옵션

| 옵션 | 타입 | 추가 가격 |
|------|------|-----------|
| 샷 추가 | 체크박스 | +500원 |
| 시럽 추가 | 체크박스 | +0원 (무료) |

**옵션 동작:**
- 체크박스 선택 시 해당 옵션이 활성화
- 담기 버튼 클릭 시 선택된 옵션과 함께 장바구니에 추가
- 옵션 가격은 장바구니에서 합산

## 4. 장바구니 영역 (Cart Section)

### 4.1 화면 위치
메뉴 목록 하단에 고정된 영역으로 표시

### 4.2 장바구니 레이아웃

```
┌─────────────────────────────────────────────────────────────────────┐
│  장바구니                                                           │
│                                                                     │
│  아메리카노(ICE) (샷 추가)  X 1      4,500원                        │
│  아메리카노(HOT)           X 2      8,000원                        │
│                                                                     │
│                                    총 금액  12,500원               │
│                                                                     │
│                                    ┌──────────────┐                │
│                                    │   주문하기   │                │
│                                    └──────────────┘                │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.3 장바구니 아이템 구성

| 요소 | 설명 |
|------|------|
| 메뉴명 | 메뉴 이름 + 선택된 옵션 표시 (괄호) |
| 수량 | X {수량} 형태로 표시 |
| 가격 | 단가 × 수량 = 소계 |
| 총 금액 | 모든 아이템 합계 |

### 4.4 장바구니 상호작용

| 동작 | 설명 |
|------|------|
| 아이템 추가 | 담기 버튼 클릭 시 장바구니에 추가 |
| 수량 증가 | 같은 메뉴+옵션 조합 재추가 시 수량 증가 |
| 주문하기 | 주문 생성 API 호출 후 주문 상세 페이지로 이동 |

### 4.5 주문하기 버튼

**스타일:**
- 배경: 테두리 스타일 (기본) / 채워진 스타일 (hover)
- 크기: 폭 120px, 높이 40px
- 폰트: 14px, 중앙 정렬

**동작:**
1. 버튼 클릭
2. 로그인 확인 (미로그인 시 로그인 페이지로 이동)
3. 주문 생성 API 호출 (POST /api/orders)
4. 성공 시 주문 상세 페이지로 이동
5. 실패 시 에러 메시지 표시

## 5. 사용자 플로우

### 5.1 메뉴 조회 및 주문 플로우

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   메뉴      │     │   옵션      │     │  장바구니   │     │   주문      │
│   목록      │ ──▶ │   선택      │ ──▶ │   확인     │ ──▶ │   완료      │
│   조회      │     │   + 담기    │     │   + 주문   │     │   페이지    │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

### 5.2 상세 플로우

1. **메뉴 조회**
   - 사용자가 메뉴 화면에 진입
   - 메뉴 목록이 그리드 형태로 표시됨

2. **옵션 선택**
   - 원하는 메뉴 카드에서 옵션 체크박스 선택
   - 샷 추가, 시럽 추가 등 선택

3. **장바구니 추가**
   - "담기" 버튼 클릭
   - 선택한 메뉴와 옵션이 장바구니에 추가
   - 장바구니 영역 업데이트

4. **주문 생성**
   - "주문하기" 버튼 클릭
   - 주문 API 호출
   - 주문 완료 후 상세 페이지로 이동

## 7. 컴포넌트 명세

### 7.1 컴포넌트 구조

```
src/
├── components/
│   ├── Navbar.jsx           # 상단 네비게이션
│   ├── MenuCard.jsx         # 메뉴 카드 컴포넌트
│   ├── MenuOption.jsx       # 옵션 체크박스
│   ├── Cart.jsx             # 장바구니 컴포넌트
│   └── CartItem.jsx         # 장바구니 아이템
├── pages/
│   ├── Menu.jsx             # 메뉴 페이지
│   └── OrderDetail.jsx      # 주문 상세 페이지
```

### 7.2 Props 정의

**MenuCard.jsx**
```typescript
interface MenuCardProps {
  id: number;
  name: string;
  price: number;
  description: string;
  image?: string;
  options: {
    name: string;
    price: number;
  }[];
  onAddToCart: (item: CartItem) => void;
}
```

**Cart.jsx**
```typescript
interface CartProps {
  items: CartItem[];
  totalAmount: number;
  onOrder: () => void;
}

interface CartItem {
  menuId: number;
  menuName: string;
  quantity: number;
  price: number;
  selectedOptions: string[];
}
```

## 9. API 연동

### 9.1 메뉴 조회

```
GET /api/menus?cafe_id={cafeId}

Response:
[
  {
    "id": 1,
    "name": "아메리카노(ICE)",
    "price": 4000,
    "description": "간단한 설명...",
    "options": [
      { "name": "샷 추가", "price": 500 },
      { "name": "시럽 추가", "price": 0 }
    ]
  }
]
```

### 9.2 주문 생성

```
POST /api/orders

Request:
{
  "cafe_id": 1,
  "items": [
    {
      "menu_id": 1,
      "quantity": 1,
      "selected_options": ["샷 추가"]
    },
    {
      "menu_id": 2,
      "quantity": 2,
      "selected_options": []
    }
  ]
}

Response:
{
  "id": 1,
  "total_amount": 12500,
  "status": "pending",
  "estimated_pickup_time": "2025-12-17T10:45:00Z"
}
```

## 11. 접근성 (Accessibility)

### 11.1 요구사항

- 모든 이미지에 alt 텍스트 제공
- 버튼에 aria-label 제공
- 키보드 네비게이션 지원
- 포커스 표시 스타일 유지
- 색상 대비 비율 4.5:1 이상

### 11.2 구현 예시

```jsx
<button 
  aria-label="아메리카노(ICE) 장바구니에 담기"
  className="focus:ring-2 focus:ring-orange-500"
>
  담기
</button>
```
